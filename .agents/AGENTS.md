# Project-Scoped AI Rules

The following rules apply to any AI assistant operating within the `opti-battery` workspace.

## 🧠 Memory & Context Management
1. **Always read `MEMORY.md` first**: At the very beginning of any new conversation or when assigned a task, you MUST read the `MEMORY.md` file located at the root of the project to understand the user's goals, the project architecture, and the current state of the development.
2. **Always update `MEMORY.md`**: After you complete any significant architectural change, new feature, or fix, you MUST update the "Memory Graph" section in `MEMORY.md` to log what was done. This ensures the next AI that assists the user has a perfect, up-to-date continuation of context.
